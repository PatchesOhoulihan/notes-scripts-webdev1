Attribute

Alle HTML-Element-Objekte bieten die Möglichkeit mit den Methoden getAttribute und setAttribute zum Auslesen und verändern von Attributewerten an.

Example Attribute auslesen: document.querySelector('tag').getAttribute('value');

Example Attribute setzen: document.querySelector('tag').setAttribute('value',8);

Example alle Attribute eines Elements zurückgeben: document.querySelector('tag').attributes
Als Rückgabewert erhält man eine NamedNodeMap, das alle Attribute und deren Werte enthält.

IDL-Attribute vs Content-Attribute

Die Methoden .getAttribute() und .setAttribute() heissen Content-Attribute, ihr Rückgabewert ist immer String.

In vielen Fällen stellt das DOM die Attribute aber auch direkt als JS-Properties auf dem HTML-Element-Objekt zur Verfügung (IDL-Attribute)
document.querySelector('tag').value
document.querySelector('tag').value=1;
IDL-Attribute können unterschiedliche Rückgabewerte besitzten.

Meisten gibt es keinen Unterschied, die IDL-Attribute greifen auf die zugrundeliegenden Content-Attribute zu.

Achtung! - boolische Attribute

Bei booleschen Attribute müssen Sei etwas aufpassen. Betrachten Sie Beispielsweise die Eigenschaft disabled. Diese Eigenschaft gibt es unterandern auf den Elementen button, input, option, select und textarea

Das IDL_Attribute hat den Type: boolean
typeof document.querySelector('button').disabled ) => boolean.
Sie können es mittels ..disabled = true oder ..disabled = false; umschalten

Arbeitet man jedoch mit den Content-Methoden: ist die Korekteschreibweise ..setattribute('disabled',"disabled")(deaktivieren) und ..setAttribute('disabled', '') zum aktivieren.

Attribute entfernen: document.querySelector('input').removeAttribute('disabled')

Auf die Fragestellung kommt es an- mit dem Value Attributen.

<input type="text" value="hallo" />

Solange sie den Text nicht ändern, sind der IDL und Content-Attribute identisch.
$('input').value => hello (IDL-Attribute)
$('input').getAttribute('value') => hello (Content-Attribute)

Trägt man jedoch einen anderen Wert in das Input feld ein:
$('input').value => world (IDL-Attribute)
$('input').getAttribute('value') => hello (Content-Attribute)
Das IDL-Attribute reflektiert die Änderung das content-attribute nicht.

NOTE: Verwenden Sie immer die IDL-Attribute, ausser Sie benötigen gezielt einen Originalwert(hardcoded bevor Benutzer änderungen durchgeführt hat.)

Der Zugriff über die Eigenschaften auf die IDL-Attribute ist aus meherern Gründen vorzuziehen:

  • Sie erhalten einen genaueren Datentype und nicht nur String
  • Sie erhalten aktuelle, möglicherweise geänderte Werte.
  • Das Verhalten ist konsistenter. Sie müssen weniger Sonderfälle beachten.
