Rauszögern, ohne abzuwarten: Defer & Async

defer verzögert die Ausführung des Scripts. Erst wenn die Seite ihr DOM komplett aufgebaut hat, führt sie das Script aus. Wird es weg gelassen wird es sofort ausgeführt ohne das der DOM des Body komplett geladen wurde

async sorgt dafür, dass der Browser die JS-Datei paralell zu anderen Dateien lädt und das ausführen des HTML-Codes an dieser Stelle nicht warten muss. Man sollte es als eine Art Performance optimierung betrachten.

Strikt und abgeschottet IIFE & strict

'use strict' lässt Javascript eine strengere Syntaxprüfung durchführen
IIFE(Immediatly Invoked Function Expression) ist ein Functionsausdruck, der sofort ausgeführt wird IIFE
Bringt 2 Vorteile mit:

  • Die Function schafft einen eigenen Scope. Variable die in ihn angelegt werden sind nicht global.
  • Ist ein 'use strict' in einer IIFE so gilt dieses nur innerhalb des IIFE und andere Teile/Bibliotheken werden nicht der stärkeren Syntaxprüfung unterzogen.

Refactoring bedeutet, das verbessern von Code OHNE dabei sein Verhalten zu verändern.

Ein wichtiges Prinzip beim Programmieren ist das Single Responsibility Principle, es bedeutet, dass jede funktion immer nur eine Sache tuen sollte.
