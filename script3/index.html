Im App-Zoo

immer mehr Anwendungen verschieben sich »ins Web«. Auf der Anwenderseite ist dann nur noch ein Webbrowser (und eine Internetverbindung) nötig, um die Anwendung auszuführen. Man spricht in diesem Fall von Webanwendung.

Webanwendungen bringen eine Reihe von Vorteilen mit sich, in erster Linie sind sie wesentlich leichter zu verteilen und zu warten. Außerdem kommt es nicht darauf an, welches Betriebssystem auf dem Rechner des Anwenders läuft (solange ein Webbrowser vorhanden ist).

die Anwendung für Mobilgeräte (Smartphones, Tablet-Computer u. a.) — kurz App genannt. Apps sind vergleichbar mit Desktopanwendungen, da sie lokal laufen.

Kennen Sie folgende Situation? Sie benutzen eine Webanwendung. Ein typischer Arbeitsablauf umfasst 3 oder 4 Klicks. Allerdings dauert es nach jedem Klick eine gefühlte Ewigkeit, bis der Rechner reagiert...

Ein Grund hierfür liegt in der Architektur klassischer Webanwendungen. Dabei wird nach jeder Benutzeraktion die gesamte HMTL-Seite der Anwendung vom Webserver neu zusammengestellt und zum Webbrowser gesendet — nicht sehr effizient!

Während dieses Vorgangs kann keine weitere Benutzeraktion stattfinden. Falls alles optimal abläuft und die Webanwendung effizient programmiert ist, dauert der gesamte Vorgang vielleicht 0,05 bis 0,5 Sekunden. Ist der Webserver aber überlastet oder die Internetverbindung nicht ausreichend schnell, kann der Wert die kritische Grenze von ca. 1 Sekunde überschreiten, ab der die Wartezeiten den Gedankenfluss des Benutzers unterbrechen und als lästig wahrgenommen werden (siehe J. NIELSEN (1993)).

Diese Wartezeiten und Verzögerungen sind ein entscheidender Nachteil klassischer Webanwendungen gegenüber anderen Anwendungen.

Wie kann man also vermeiden, dass jede Benutzeraktivität das Neuladen der Seite vom Server nötig macht? Ganz einfach: Zumindest Teile der Anwendung müssen direkt im Webbrowser ausgeführt werden!

Es gab in der Vergangenheit einige Ansätze, um das Problem zu lösen:

  • das 1995 erschienene Java-Applet Plug-in, das es erlaubt, Anwendungen in der Programmiersprache Java im Webbrowser auszuführen
  • das kurz darauf von Macromedia (heute Adobe) entwickelte Flash Player Plug-in mit dem Schwerpunkt multimediale Anwendungen (Animationen, Audio, später auch Video)
  • und natürlich JavaScript, ebenfalls bereits im Jahre 1995 damals noch unter anderem Namen vorgestellt und später als ECMAScript standardisiert.

Mittlerweile (und besonders in Verbindung mit HTML5) hat sich JavaScript als meistverbreitete Technologie etabliert. Der Flash Player besetzt noch eine starke Nische im Bereich Video und Spiele, Java-Applets werden größtenteils nicht mehr eingesetzt (Gegenbeispiel: Teile von ELSTER (elektronische Steuererklärung in Deutschland) sind als Java-Applet umgesetzt).

Ein Trend der letzten Jahre ist, große Teile einer Webanwendung im Webbrowser laufen zu lassen: HTML, CSS, Bilder und JavaScript-Code werden nur zu Beginn vom Server geladen und dann während der gesamten Benutzersitzung nie mehr neu geladen. Die Rolle des Servers beschränkt sich im Wesentlichen darauf, Daten, die vom Webbrowser im Hintergrund gesendet werden, zu verarbeiten und zu speichern! Eine solche Webanwendung bezeichnet man als Single-Page-Application (wörtlich »Einzelseiten-Anwendung«).

ES2015 Unterstützung

Klassen

Die Sprache unterstützt mit ES2015 nun das aus anderen Programmiersprachen bekannte Schlüsselwort class.
Example:

                        
                            class Buch {
                                constructor(titel, autor) {
                                    this.titel = titel;
                                    this.autor = autor;
                                }
                                toString() {
                                    return '»' + this.titel + '« von ' + this.autor;
                                }
                            }
                            
                            //Aufruf
                            var b = new Buch('Per Anhalter durch die Galaxis', 'Douglas Adams');
                            console.log(b.toString());
