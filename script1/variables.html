4 Variablen oder die Realität in Schubladen

Variablen anlegen

let username = "Oswine";

Schlüsselwörter

Wie Sie nun schon wissen, handelt es sich bei username um einen Variablenbezeichner und bei der Anweisung let username; um eine Variablendeklaration. Das Wort let dagegen ist ein sogenanntes Schlüsselwort (engl. keyword). Schlüsselwörter sind ein fes- ter Bestandteil der Programmiersprache. Sie haben eine fest definierte Bedeutung und können nicht für andere Zwecke eingesetzt werden. Sie werden in JS übrigens immer komplett kleingeschrieben.

Variablen Werte zuweisen

Um einen Inhalt in die Variable »hineinzupacken«, gibt es in JavaScript das Konzept der Zuwei- sung – Sie weisen einer Variable ihren Inhalt zu. Ausgedrückt wird die Zuweisung in JS durch den sogenannten Zuweisungsoperator — dargestellt durch das Gleichheitszeichen = . Ebenfalls essenziell sind die Anführungszeichen. Daran erkennt JS, dass es sich um den Wert Ladislaus handelt und nicht etwa um eine Variable, die Ladislaus heißt.

Ist Ihnen der Inhalt der Variable schon beim Erstellen der Variable bekannt, können Sie die Schreibweise abkürzen — mittels einer sogenannten Variableninitialisierung.

Jetzt aber prompt bitte!

In der Praxis möchten Sie natürlich auch Daten verwenden, die Sie nicht vorher selbst bestimmen, sondern solche, die ein Anwender eingibt.

let username = prompt("Please let me know your name!");

Grundsolide:Konstanten

Ein Konstante ist einer Variable recht ähnlich. Der Unterschied besteht nur darin, dass sie nicht variabel, sondern konstant ist

const VAT_RATE = 1.20;

Damit dienen diese Konstanten gewissermaßen der Konfiguration Ihres Programms. Wenn Sie das konsequent umsetzen, finden Sie am Anfang ihres Programms eine Ansammlung die- ser Konstanten und sehen somit auf einen Blick alle relevanten »Stellschrauben«.

Programmierrichtlinien ➤ Vermeiden Sie Magic Numbers. ➤ Schreiben Sie Konstanten, die der Konfiguration dienen, kom- plett in Großbuchstaben und verwenden Sie den Underscore _ zur Worttrennung. ➤ Deklarieren Sie Konstanten, die der Konfiguration dienen, am Anfang Ihres Codes.

Kampf der Giganten: Variable vs. Konstante

Ein Variable kann ihren Wert jederzeit ändern. Bei einer Konstanten ist das nicht möglich. Sobald sie eine Konstante einmal auf einen Wert festgenagelt haben, bleibt der Wert bis zum Ende des Programms fest. Hier ein Gegenbeispiel:

"use strict";
2 const VAT_RATE = 1.20;
3 VAT_RATE = 1.25;

Der JS-Interpreter quittiert diesen Code mit einer Fehlermeldung wie »VAT_RATE is read-only« und bricht das Programm ab.
